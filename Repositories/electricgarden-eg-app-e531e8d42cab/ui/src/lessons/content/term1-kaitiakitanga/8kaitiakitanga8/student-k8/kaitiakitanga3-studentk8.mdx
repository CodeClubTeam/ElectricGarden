### What is **debugging**?

<Captioned>
    He aha <b>te patuiro</b>?
</Captioned>

<br />

##### In the garden a bug is an **insect**, but in programming it means something a bit different!

##### In programming, a **bug** is an **error** in our code, or a **mistake** that prevents a program from doing what it is supposed to do.

<br />

<Captioned font={'Comfortaa'} size={'25'}>
    <b>bug</b> = an error, or mistake, that stops a program from working
    properly
</Captioned>

<br />

##### For example, we would consider it a bug if we found our instructions were **out of sequence**, if they were **missing a step**, or if a step was **incorrect**.

##### When creating a program, it is important to use **logic** and **test** each process to look for any bugs and fix them if they are found. This is called **debugging**!

<br />

<Captioned font={'Comfortaa'} size={'25'}>
    <b>debugging</b> = looking for and fixing errors in a program
</Captioned>

<br />

##### So, rather than managing insects in the garden, debugging is managing errors in our programming.

##### **Debugging** is an important practise to help make sure our programs work the way they were designed.

<br />
